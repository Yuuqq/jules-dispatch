THE VISION

Google Jules 很强大。
但它本可以更强大。

默认情况下，Jules 一次只能处理一个任务。
当你需要重构整个代码库、批量迁移框架、或为 20 个模块添加功能时，串行执行会成为瓶颈。

以前的体验
  • 手动重复编写 prompt
  • 等待一个任务完成才能开始下一个
  • 难以规模化处理大型重构
  • AI 代理无法自主并行调用 Jules
使用 jules-dispatch 之后
  • 一句话描述目标，AI 自动拆解为多个并行任务
  • 一键启动 10~50 个 Jules 会话
  • 实时监控、批准计划、发送消息、取消失控任务
  • Claude / Codex 可以直接把 Jules 当作工具使用

v1.2 核心新功能

停止手写任务。
让 AI 为你规划。

给 jules-dispatch 一句话，它就能调用你配置的 LLM（OpenAI / Ollama / Groq / OpenRouter 等），智能拆解成多个可并行执行的 Jules 任务。

示例意图：

jules-dispatch auto "将项目中所有 Express 路由迁移到 Fastify，并为每个路由添加请求验证测试"

它会规划出 6 个独立任务（认证路由、用户路由、计费路由、中间件替换、启动文件更新、测试），然后询问是否立即并行调度。

支持的 LLM 提供商
OpenAI（默认）
Ollama（本地）
Groq
OpenRouter
Together / Fireworks / DeepInfra / Azure OpenAI / LiteLLM 等任意兼容端点
通过环境变量或命令行参数灵活配置，无需修改代码。
GET STARTED IN 4 MINUTES

四步上手，立即释放并行生产力

01
全局安装
一次安装，永久可用
npm install -g jules-dispatch
02
初始化配置
连接你的 Jules API Key
jules-dispatch init

交互式向导配置 JULES_API_KEY。支持 --api-key 按次传入。

03
声明任务（YAML）
版本可控，可团队协作

创建 tasks/ 目录，使用简单 YAML 定义任务（支持多文档 ---）：

# tasks/migrate-to-fastify.yaml
title: "迁移认证路由到 Fastify"
repo: "your-org/your-repo"
branch: "main"
prompt: |
  将 /api/auth 下的所有 Express 路由迁移到 Fastify。
  使用 Fastify 插件系统，保持原有中间件逻辑。
  添加适当的 schema 验证。
04
并行调度执行
启动你的 AI 编码军团
单任务 / 批量 / AI 规划
jules-dispatch dispatch tasks/migrate-to-fastify.yaml
jules-dispatch batch tasks/ --parallel 12
jules-dispatch auto "为整个项目添加完整的 TypeScript 类型和 Vitest 测试"
查看完整命令参考与高级选项
